\section{Automated cleaning engine}\label{sc:cleaning_engine}
The core challenge in converting heterogeneous variable types across the six datasets (Table~\ref{tab:reduced_dataset_shape}) was achieving a single numerical standard suitable for downstream analysis. This required enforcing \emph{numerical homogeneity}, defined here as representing all retained features using a single numerical representation (\texttt{float64}) with a unified missing or non response sentinel ($\texttt{-1.0}$). 

Categorical variables were dominant ($2{,}916$ columns) while numeric types were fragmented, with \texttt{float64} ($462$ columns) adopted as the common standard to support decimal values from midpoints, ordinal scaling, and frequency normalisation. The solution was implemented as two stages: automated \emph{variable scenario detection} function (\texttt{detect\_column\_scenario\_mapping}, (Appendix~\ref{app:variable_mapping_code})) and a deterministic \emph{six step numerical cleaning engine} (Appendix~\ref{app:cleaning_pipeline_code}). Nine scenario categories were defined initially and three served only as validation fallbacks and were eliminated as coverage improved, leaving six active transformation steps applied uniformly across all waves (Appendix~\ref{tab:cleaning_tables}).

\subsection{Stage 1: Variable scenario mapping}
All variables passed sequentially through the six step pipeline, most were resolved in Steps 1--5, with step 6 (Appendix~\ref{app:step6_code}) handling only residual variables that could not be resolved by earlier transformations.

To ensure robustness, the classification output was immediately processed by a validation routine that generated a comprehensive dictionary of variable states before transformation and feature representations after, including all representative sample values for each variable. This allowed direct inspection of how the same variable was represented across waves prior to cleaning, particularly in cases where identical semantic categories appeared in different orders or formats. By comparing these grouped samples before and after each cleaning step, classification errors and inconsistent mappings were identified and corrected prior to execution of subsequent transformations.

The dictionary generated inside the \texttt{show\_grouped\_vars\_for\_scenarios} function (Appendix~\ref{app:md_script}) was formatted to create an automatic friendly ``.md'' file for rapid, surface level verification, which honed the final mapping dictionary (an example can be found in Appendix~\ref{app:md_audit}) for all features across all waves. This process of using automated detection followed by visual, manual validation was essential for correcting errors in classification logic before the subsequent scenario transformations were executed in the pipeline.

\subsection{Stage 2: Deterministic six step cleaning sequence}\label{sc:stage2}
The nine input scenarios were processed sequentially by the \emph{six step cleaning engine}, resolving simple and unambiguous cases first to maximise the proportion of successfully transformed variables before tackling more complex structures, particularly censored and multi-class scenarios (Figure~\ref{fig:cleaning_flow}). With later steps progressively shifting from cleaning into semantic feature engineering, this deterministic sequence applied identical transformation and feature encoding logic to each wave, with later steps (particularly Steps 4--6) performing rule based feature engineering by deriving semantically meaningful numerical representations directly from observed category tokens at runtime. The rationale for this embedded feature engineering approach is detailed in Section~\ref{subsec:feature_engineering}, implementation is provided in Appendix~\ref{app:step6_code}.
This design reflects the empirical diversity of survey encodings: a small subset of variables only reveals its structure once simpler transformations have been applied.

The final cleaning pipeline consisted of six deterministic steps applied uniformly across all \gls{TILDA} waves, with increasingly complex transformations handled in later stages. Steps 4--6 required particular attention due to hybrid, ordinal, or multi-class formats. A summary of each step and examples is provided below.

\begin{enumerate}
    \item \textbf{Empty / Blank Values:} All missing entries, empty strings, and whitespace were converted to the standard non-response sentinel ($\texttt{-1.0}$).
    
    \item \textbf{Sentinel Values:} Wave specific non-response codes (e.g., $\texttt{-98}$, $\texttt{-99}$) and textual sentinels (e.g., \texttt{``dk''}, \texttt{``not applicable''}) were mapped to the unified ($\texttt{-1.0}$) placeholder.

    \item \textbf{Numeric Transformation:} Numeric like columns containing embedded units, symbols, or hybrid strings (e.g., $\texttt{50+}$, $\texttt{<10}$, $\texttt{1-2.5}$) were parsed using regex and converted to \texttt{float64}. For ranges, midpoints \&/or boundary values were used. For censored or annotated values, appropriate upper/lower bounds were applied. 

    \item \textbf{Censored / Ordinal Variables:} This step addressed variables representing ordinal scales, bracketed numeric ranges, or frequency expressions (e.g., ``2--3 days a week'', ``daily''). Transformations included:
    \begin{itemize}
        \item[-] Converting textual ordinal labels and Likert scale responses to numeric equivalents.  
        \item[-] Mapping calendar based frequency descriptors to numeric weekly equivalents.  
        \item[-] Manual verification was conducted for low count or unusual variables using the Markdown audit outputs to confirm semantic meaning was preserved.  
    \end{itemize}

    \item \textbf{Binary Categorical Variables:} Two level features (e.g., Yes/No, Male/Female) were dynamically encoded using mappings derived from the observed value set of each variable:
    \begin{itemize}
        \item[-] Semantic rules were applied e.g., (``no'', ``not'', ``female'' $\to 0.0$) and (``yes'', ``male'' $\to 1.0$).  
        \item[-] Count based detection verified that variables had exactly two non-missing unique values.  
        \item[-] Features with more than two sample values were also analysed for sentinel presence:
        \begin{itemize}
        \item[-] If a variable had three observed values and one was a sentinel, it was treated as binary.  
        \item[-] If a variable had four observed values and two were sentinels, it was treated as binary.  
        \end{itemize}
        \item[-] Sentinels were preserved as $\texttt{-1.0}$.  
        \item[-] This approach ensured order invariant encoding across waves, even when original labels or numeric prefixes differed.
        \item[-] \textit{Example (sentinel case):} Feature \texttt{X} with pre-transformed values \texttt{[`NOT Getting in or out of bed', -1.0, `Getting in or out of bed']} was encoded as \texttt{[0.0, -1.0, 1.0]}, demonstrating how a sentinel preserves binary semantics.
    \end{itemize}

    \item \textbf{Multi-class / Mixed Variables (rule based feature engineering):} Variables with remaining multi-class structure, mixed types, or unresolved string numeric hybrids after earlier transformations were processed in the final step.

    Technical implementation for this step are provided in Appendix~\ref{app:step6_code}:
    \begin{itemize}
        \item[-] Text based parsing extracted numbers, ranges, units, percentages, and frequency descriptors into numeric values, yielding interpretable numerical feature representations.  
        \item[-] Sentinel handling ensured all known non-response codes mapped to ($\texttt{-1.0}$).  
        \item[-] Original values were grouped and assigned numeric identifiers in first seen order, non numeric categorical labels were assigned integer codes above the highest observed numeric value to prevent collisions.  
        \item[-] Manual inspection using the Markdown audit file verified that semantic equivalence was maintained across waves.  
        \item[-] Step 6 was designed to be \emph{order invariant}, dynamically interpreting category values within each wave rather than relying on predefined lookup tables shared across waves. This preserved consistent numeric representation regardless of variable ordering or wave specific encodings.
        \begin{itemize}
        \item[-] \textit{Example:} Step 5 handled simple binary variables, e.g., (any pain reported) with pre transformed values \texttt{[`pain', `No pain']} and numeric mapping \texttt{[1.0, 0.0]}. In contrast, step 6 dynamically processed multi-class variables, e.g., (pre tax wage/salary) many of which contained over 20 distinct pre transformed categories (e.g., \texttt{-1.0, `200--299', `Less than 100', \ldots, `6,000--6,999'}) into consistent numeric representations (e.g., \texttt{-1.0, 249.5, 99, \ldots, 6499.5}). This illustrates why step 6 required wave specific, dynamic parsing to maintain semantic equivalence across diverse observed values.
        \end{itemize}
    \end{itemize}
\end{enumerate}

\subsection{Embedded feature engineering}\label{subsec:feature_engineering}
Although the six step pipeline is framed primarily as a numerical cleaning procedure, steps 4--6 also perform explicit rule based feature engineering by interpreting heterogeneous categorical survey responses and converting them into semantically meaningful numerical representations. Rather than acting as a separate post processing stage, feature engineering is embedded directly within the cleaning logic, formalising ordinal structure, frequency, and magnitude encoded in text into directly modelable \texttt{float64} values suitable for downstream analysis.

Crucially, this process engineers the representation of each feature rather than expanding the feature space. Free text frequency responses such as ``\texttt{once or twice a week}'', ``\texttt{a couple of times per month}'', or ``\texttt{almost every day}'' are not treated as nominal categories. Instead, lexical cues (e.g., \texttt{once}, \texttt{twice}, \texttt{couple}) are resolved to numeric quantities and normalised onto a common temporal scale, yielding a continuous rate representation. Implicit semantic information such as ordering, intensity, and frequency is therefore preserved through deterministic parsing and normalisation without introducing additional variables. This design maintains the original dimensionality of the dataset while ensuring numerical homogeneity and semantic integrity, producing a compact and model ready representation of complex real world survey data.

\subsection{Cross wave semantic verification}
Given the absence of guaranteed category ordering across waves, preserving \emph{semantic equivalence} during numerical encoding and feature representation was essential to avoid introducing distortions into downstream \gls{ML} modelling. Pre and post transformation values for each group were exported to a readable Markdown (\texttt{.md}) file. This representation confirmed that features of known importance for downstream predictive models were preserved and correctly encoded. Self-rated general health, comparative self-rated health, and smoking frequency are among the most influential predictors in \gls{FI} based mortality models~\cite{moguilnerImportanceAgePrediction2021}. Retaining these features from the manual filtering thresholds applied in Section~\ref{sc:manual_var_filtering} and encoding them consistently across waves ensured that the cleaned dataset maintains coverage and semantic integrity, preventing distortions that could potentially compromise model performance.

A key challenge in cleaning the five \gls{TILDA} waves was the lack of guaranteed ordering for categorical values of the same variable across different waves. Table~\ref{tab:wave_ordering_example} illustrates this for waves 4 and 5.

Gender (\texttt{sex}) variables appeared under different value encodings, with varied category orderings and numeric prefixes across waves. Values such as \texttt{[`Male', `Female']}, \texttt{[`Female', `Male']}, and prefixed forms like \texttt{[``1.male', `2.female']} were all mapped to a consistent numeric representation (\texttt{1.0} for male, \texttt{0.0} for female). This ensured that semantic equivalence, rather than positional ordering or wave specific encoding, determined numerical assignment.

The Markdown audit confirmed that semantically identical categories were correctly grouped across waves regardless of variable name or encoding order, and that complex value spaces (e.g., income bands, frequency descriptors) were being collapsed to stable numeric representations consistently across waves. Full audit examples are provided in Appendix~\ref{app:md_audit}.

\section{Validation of the transformed features}
Validation assessed whether the pipeline preserved meaningful biological structure by confirming that known relationships reported in the literature were retained in the cleaned, transformed data.

Due to restricted access to certain outcome variables within \gls{TILDA}, full end-to-end replication of all published models was not always feasible. In particular, access through the hotdesk facility for the official mortality outcomes is subject to additional governance requirements, including institutional \gls{GDPR} training certification and approval by the \gls{TILDA} data access committee~\cite{HotDesksIrish}. 

As a result, validation focused on two complementary strategies: 
\begin{enumerate}[label= (\roman*)]
\item Internal consistency checks against established biological and clinical relationships, and 
\item Constrained replication of published modelling frameworks using available predictors and the derived \emph{8-year mortality proxy} outcome. 
\end{enumerate}
Together, these validation steps demonstrate that the cleaning pipeline retains expected relational patterns and produces datasets appropriate for the downstream modelling tasks presented in Chapter~\ref{ch:chapter5}.

\subsection{Internal feature relationships}
The first validation stage assessed whether cleaned predictors preserved biologically and clinically plausible relationships reported in prior \gls{TILDA} studies. This analysis focused on predictors identified by~\cite{davisLinearRegressionbasedMachine2021} as accounting for approximately $95\%$ of the explained variance in \gls{GSR}, defined as the difference between maximum and \gls{UGS}.

As \gls{GSR} is not directly available in the public wave 3 dataset, validation was performed by examining the directional relationships between the 7 primary predictors themselves. This approach evaluates whether the encoding process, including sentinel handling, missing value treatment, and ordinal mapping, preserved the expected structure of the data rather than simply fitting to a particular outcome.

Figure~\ref{fig:heatmap} presents the pairwise Pearson correlation matrix for the selected predictors after the deterministic six step cleaning pipeline. Grip strength exhibited a negative association with both age and chair stand time, reflecting the inverse relationship between muscular strength and physical performance decline. Cognitive performance (\gls{MOCA}) demonstrated a moderate negative correlation with age and a positive association with educational level, consistent with known patterns of cognitive reserve in older adults. Education correlated positively with cognitive performance, and \gls{BMI} showed the expected weak negative association with age, reflecting reduced physiological reserve in older cohorts.

Table~\ref{tab:directional_validation} summarises the expected and observed directional relationships between key predictor pairs. Notably, grip strength showed moderate negative correlations with both age ($r=-0.265$) and chair stand time ($r=-0.170$), reflecting the expected inverse relationship between muscular strength and physical performance decline. Cognitive performance (\gls{MOCA}) displayed a moderate negative association with age ($r=-0.381$) and a positive association with education ($r=0.367$), consistent with known patterns of cognitive reserve in older adults.\ \gls{BMI} demonstrated the anticipated weak negative correlation with age ($r=-0.073$). These results confirm that the cleaning pipeline retained the expected numerical structure and relational patterns of the predictors, rather than introducing artificial associations.

\subsection{Model based validation}
To evaluate whether the cleaned dataset preserves predictive utility, a constrained replication of~\cite{moguilnerImportanceAgePrediction2021} was conducted using available wave~1 predictors. The study predicted 8-year mortality using 32 \gls{FI} items. In the cleaned wave 1 dataset following the deterministic six step cleaning pipeline described in Section~\ref{sc:stage2}, only 16 of these 32 \gls{FI} items remained following the manual variable filtering described in Section~\ref{sc:manual_var_filtering}, which prioritised global numeric consistency across waves.

An \emph{8-year mortality proxy} outcome was constructed to approximate the sex specific 8-year mortality rates reported in~\cite{moguilnerImportanceAgePrediction2021} (men: $14.9\%$, women: $11.1\%$), derived from linkage to Ireland’s Central Register Office records~\cite{wardLinkingDeathRegistration2020a}. Due to access restrictions, the official mortality data were unavailable. Therefore, mortality status was first assigned by randomly selecting deceased participants from the wave 1 cohort aged 50 years or older to match the published sex specific mortality proportions observed at wave 5 follow up. An equal number of surviving participants were then randomly sampled within each sex to construct balanced datasets for model training. The resulting random proxy datasets included 1,114 men (557 deceased) and 982 women (491 deceased), reflecting the baseline wave 1 population aged $\ge50$ years ($n = 8,172$: 3,743 men and 4,429 women).

Two variants of this proxy were implemented:
\begin{enumerate}[label= (\roman*)]
\item the purely random assignment described above, used as a structural baseline, and 
\item biologically informed probabilistic assignment based on latent \gls{FR} structure extracted via \gls{PCA}. This dual construction allowed separation of statistical artefact from biologically meaningful signal. 
\end{enumerate} 

The random proxy serves as an initial diagnostic benchmark. Mortality labels were assigned by randomly selecting deceased participants within each sex to match the published 8-year mortality proportions. While this preserves overall class balance, it does not encode any biological or \gls{FR} related information and therefore serves only as a structural validation baseline.

For the biologically informed proxy, the 16 available \gls{FI} items were standardised to zero mean and unit variance using z-score transformation.\ \gls{PCA} was then performed separately within each sex on complete case observations to extract the principal component representing the dominant latent \gls{FR} structure. The \gls{PC1} was retained as a latent \gls{FR} score, and its direction was aligned such that higher \gls{PC1} scores correspond to higher \gls{FR} by enforcing a positive correlation with the calculated \gls{FI}. This latent \gls{FR} score was subsequently combined with age to construct a probabilistic mortality assignment reflecting both biological vulnerability and time dependent risk.

Two model variants were tested for each sex:
\begin{enumerate}
    \item \textbf{\gls{FI} only model:} using the 16 available \gls{FI} items, without age.
    \item \textbf{\gls{FI} $+$ Age model:} including age as an additional predictor.
\end{enumerate}
Model training and evaluation followed the procedure reported in~\cite{moguilnerImportanceAgePrediction2021} as closely as permitted by the available data. Features were min-max normalised prior to modelling. For each sex, data were split into training $(80\%)$ and testing $(20\%)$ sets. To mitigate overfitting and obtain robust estimates of model performance, 5-fold stratified cross-validation was applied on the training data.\ \gls{AUC} was selected as the primary evaluation metric due to its widespread use in prior \gls{TILDA} and \gls{ML} based studies (Figure~\ref{fig:prisma}) and its robustness to class imbalance.

For six of the sixteen \gls{FI} items, the cleaned dataset transformations differed from the scoring rules published in~\cite{moguilnerImportanceAgePrediction2021}. To align precisely with the original 16/32-item \gls{FI} methodology, these items were recoded following the published ordinal and dichotomous mappings. This step was necessary to ensure that the six remapped \gls{FI} items reflects the same interpretation of health deficits used in the literature and is consistent with standard \gls{FI} construction principles~\cite{searleStandardProcedureCreating2008}. Models were then re-evaluated using this partial remapping \gls{FI} to assess sensitivity to feature encoding.

The resulting \gls{AUC} values are reported separately for (i) originally cleaned (Figure~\ref{fig:roc_cleaned_fi_pca}) vs 6/16 recoded \gls{FI} items (Figure~\ref{fig:roc_recoded_fi_pca}), (ii) random vs \gls{PCA} informed mortality assignment, (iii) \gls{FI} only vs \gls{FI} $+$ Age models, and (iv) men and women.

These results highlight several key points:
\begin{enumerate}
\item \textbf{Random proxy baseline (cleaned vs recoded \gls{FI}):}  
Under purely random mortality assignment, discrimination remained at chance level for both versions of the \gls{FI}. For the cleaned 16 \gls{FI} items, the \gls{FI} only model achieved \gls{AUC} = 0.507 (men) and 0.512 (women), increasing marginally to 0.516 (men) and 0.531 (women) when age was included. When six of the sixteen available deficits were recoded to align with the published \gls{FI} scoring rules~\cite{moguilnerImportanceAgePrediction2021}, the \gls{AUC} values increased to 0.567 (men) and 0.537 (women) for the \gls{FI} only model, and to 0.572 (men) and 0.605 (women) for the \gls{FI} $+$ Age model. Despite this increase relative to the fully cleaned encoding, discrimination remained close to chance under random label assignment.

These results confirm that, without embedding biological \gls{FR} structure into the outcome, predictive performance reflects statistical noise rather than meaningful signal, regardless of encoding refinement.
\item \textbf{\gls{PCA} informed mortality proxy (cleaned vs recoded \gls{FI}):}  
When mortality assignment incorporated the \gls{PCA} derived \gls{FR} representation \gls{PC1}, discrimination improved relative to the random baseline for both \gls{FI} constructions. For the 16 cleaned \gls{FI} items, the \gls{FI} only model achieved \gls{AUC} = 0.591 (men) and 0.667 (women), increasing to 0.602 (men) and 0.660 (women) when age was included.
When six of the sixteen deficits were recoded in accordance with the published scoring framework, the \gls{FI}-only model achieved \gls{AUC} = 0.636 (men) and 0.633 (women), while the \gls{FI} $+$ age model achieved 0.643 (men) and 0.652 (women).

This indicates that closer alignment with published ordinal deficit scoring improves discrimination relative to the simplified numeric encoding applied during initial cleaning.
\item \textbf{Comparison with published performance:}  
Absolute \gls{AUC} values remain below those reported in~\cite{moguilnerImportanceAgePrediction2021} (approximately 0.70 for \gls{FI} only and up to 0.80–0.90 when age is included), which is expected given the use of a constructed mortality proxy and only 16 of the original 32 \gls{FI} items. However, the methodological sequence reported in the original study is preserved:
\begin{enumerate}[label= (\roman*)]
\item Random assignment yields chance level performance, 
\item Extracting latent \gls{FR} structure via \gls{PCA} improves discrimination, 
\item Inclusion of age generally improved mortality prediction. 
\end{enumerate}
The cleaned dataset maintains realistic sex distributions (45.8\% men, 54.2\% women) and mortality proportions consistent with published estimates, confirming that preprocessing preserved demographic and structural integrity required for downstream modelling.
\end{enumerate}
